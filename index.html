Lesson 1: The Basics of the Web and HTML

Thinking Like a Programmer

Programmers and coders use procedural thinking - taking a big problem and breaking down the solution into a set of steps - in order to write valid and useful code. Some examples of the types of thinking involved are the following:

  • Abstract thinking - 1+1=2
  • Systematic debugging - systemic way of solving problems
  • Technological empathy - thinking the way a computer does - looking at code and imagining what it looks like in a webpage, and looking at a webpage and seeing code
  • Systems thinking - taking one big idea and breaking it down into component parts (other 3 mentioned, above)
Computers Are Stupid

Programmers need to write the way a computer understands. You need to know the correct “syntax.”

Why? Because the computer can’t read your mind, it can only interpret what it sees in the code. If you forget to end a tag with a closing element, for example, forget to end an “em” tag, then it will assume that you want the rest of the narrative to be italicized.

Programmers Shouldn’t Remember Everything

This idea just highlights the fact that programmers, in this day of readily available information on the internet, shouldn’t feel that they need to memorize everything like in the past. You can just look it up. It’s more important to understand basic concepts.

The Web and its Major Components

Invented 1990’s - now has 30 billion webpages, it's basically a collection of HTML documents (HTML = Hypertext Markup Language). HTML language is what glues everything together on the web - text, videos, images. The links between webpages is what makes the web “weblike.”

The web is basically made up of three components: You, your computer and your browser; the internet, and servers. The browser makes requests via the internet to servers with a protocol called HTTP, or Hypertext Transfer Protocol. The server responds with files that your browser displays - browsers include IE, Chrome, Firefox. Servers are computers that are optimized for hosting files instead of browsing.

HTML: Markup, Tags, Elements, Attributes and Structure

HTML docs are the heart of the web and are made up of:

  • Text content - what you see
  • Mark up - what it looks like or how it is arranged (not seen)
  • References to other docs (images, videos)
  • Links to other pages

HTML markup: HTML markup is made up of tags, which are constructed like this: <opening tag>content</closing tag>. Examples:

  • <b> tag - makes content bold
  • <em> tag - means emphasis - makes content italicized
  • <a> tag - anchor tag - this is for making links
HTML attribute tags: An HTML attribute is a modifier of an HTML element. They are usually added to a HTML start tag, which along with content and end tag comprises most HTML elements. Particular attributes are only supported by particular element types; if they are added to elements that do not support them they will not function (from the HTML attribute Wiki page). Examples:
  • <a href = “www.reddit.com”>derp </a> = derp
  • <a href = “www.w3c.com”> HTML </a> = HTML
  • <a href = “www.udacity.com”>my favorite </a> = my favorite

Image tag: <img> is used to download images. For example: <img src = “url” alt = “text”>. “Url” is for the image to download. “Text” is for broken requests - the text that is displayed if image does not load. The image tag does not have or need a closing tag - it’s a “void tag” - it has no content.

Whitespace: When you enter text in multiple lines, in HTML it renders it in single space. <br> is to create a space break. This is another example of a void tag. To make double spaces you can use either a <br> or a <p> tag. To use <br> as a double space, you just use two in a row: <br> <br> <p> is a “block text” - you need a closing tag: <p> narrative </p>

Inline and block elements: "br" is an “inline element” - just ends a line. "p" narrative is a “block element” - it makes an invisible box to put narrative. The box has width and height.

Span and Div Elements: The "Span" and "Div" tags contain their content: Span is inline and is written like this: <span> text </span>. Div is block and is written like this: <div> text </span>. Unlike **div**, which is formatted with line breaks, the **span** element does not have any automatic formatting. In addition to being a block element, **div** also allows you to put footnotes on the page with a **div** element to indicate that everything inside of that div is "footnote material". And you could also style a **div** element by adding a specific class or ID to it.

Lesson 2: Creating a Structured Document with HTML

Creating a Webpage From a Graphic Design

Imagine you were handed a web design from your company’s graphic designer. You have to take this and translate it into an actual website. How do you do this? You first need a conceptual understanding of website design and how browsers retrieve and recreate webpages.

Webpage Design Basics

A house has frame, style and interactive elements. Webpages use different languages to build these elements.

  • The frame created by HTML (HyperText Markup Language) is like the house structure. Where are the walls placed? What room is your living room, bedroom? Kitchen?
  • Cascading Style Sheets or CSS language is like the house style - what color are the walls? What kind of carpeting? What decorations?
  • The interactive elements created by JavaScript (JS) is like your garage door opener or TV remote control lets you interact with your house, JS lets you interact with your webpage
How Browsers Recreate Websites Using Languages and the DOM

Let’s say you build a website, which is made up of a number of webpages built with the languages mentioned above - essentially a collection of one or more HTML documents that sit in your computer. And let’s say somebody want to access these webpages. The person’s browsers uses Hypertext Transfer Protocol, or HTTP to “request” your webpage (see “Basics of the Web, above.) Once it retrieves this information, it has to recreate this webpage in the computer. How does this happen? Via the Document Object Model, or DOM.

DOM: The Webpage’s Tree-like Structure

Here’s a trick: Using Google Chrome, select Developer Tools and mouse over the document. You can see that the browser interprets each section of the webpage as a a branching structure with sideways triangles which have opening and closing tags.

This structure is called the “Document Object Model.” It looks very much like the actual HTML code retrieved but it more than that. The DOM, according to 3WC, is a set of rules for representing and interacting with objects in HTML and other markup languages like CSS. DOM takes the retrieved code and interprets it so that the browser can understand how to “recreate” your webpage in the other computer.

How does this happen? The browser uses DOM’s rules of how to interact with the HTML, CSS and JS code to classify page content: HTML classifies what is text and what is image, and CSS classifies style.

Lesson 3: Adding CSS Style to HTML Structure

Styling and Avoiding Repetition with CSS

Cascading Style Sheets (CSS) allows you to use a specific coding syntax and rules to change how elements look on the page: size of font, color, background, order, position. CSS also helps programmers avoid repetition by enabling the separation of the web "structure" code (HTML) from the "style" code (CSS).

It does so by labeling each element you want to style in HTML, and then enabling you to link these elements to the CSS style code in a separate file. Since the style code is physically separate from the structure code, you can then change the CSS without touching the HTML.

This is an essential element of coding called DRY: "Don't Repeat Yourself."

Boxes, Boxes Everywhere

Everything on a webpage is a box, even images of circles. When you click on circles with developer tools, you see a box surrounding the circle. When you go to style, you an change the pix, color and see the result.

How to Draw Boxes: Start with a web design and choose the big box that is whole page and divide it into smaller boxes, you can use a pencil to do so. You define these boxes with a “div” tag. Each box equals different div tags.
